Module 1

Technical SEO & Site Architecture

Moving beyond keywords and content to understand how search engines crawl, render, and index our web properties.

1. Crawling & Indexing Diagnostics

Understanding the fundamental relationship between your site's architecture and the crawler budget.

  • robots.txt best practices: Knowing when to Disallow vs when to use a noindex tag.
  • XML Sitemaps: Dynamic generation, index sitemaps, and keeping error rates below 1%.
  • Google Search Console: Advanced usage of the Crawl Stats report and log file analysis basics.
Example robots.txt
User-agent: *
Disallow: /wp-admin/
Disallow: /*?filter=*
Allow: /wp-admin/admin-ajax.php

Sitemap: https://example.com/sitemap_index.xml

2. Core Web Vitals (CWV)

Performance is a ranking factor and a conversion multiplier. Learn how to diagnose CWV issues.

  • LCP (Largest Contentful Paint): Image optimization, preload scanners, and server response times (TTFB).
  • INP (Interaction to Next Paint): Diagnosing long JavaScript tasks blocking the main thread.
  • CLS (Cumulative Layout Shift): Reserving space for images and ads, and understanding font loading (FOUT/FOIT).

3. Structured Data (JSON-LD)

We do not rely on plugins for complex schema. You must know how to write and validate JSON-LD manually.

Tools to use: Rich Results Test & Schema.org documentation.

4. Migrations & Redirect Mapping

How to use Regular Expressions (Regex) to map thousands of URLs cleanly without creating redirect chains.

Module 2

Advanced Analytics & Tracking

Data integrity is our responsibility. Learn to deploy, test, and manage tracking architecture using GTM and GA4.

1. Google Tag Manager (GTM) Mastery

Stop relying on developers to add basic tracking pixels. GTM is our command center.

  • The Data Layer: Understanding `dataLayer.push()` and how to extract e-commerce data.
  • Custom HTML & JavaScript Variables: Writing small scripts to grab DOM elements.
  • Trigger Groups & Sequencing: Ensuring consent is granted *before* firing marketing tags.

2. GA4 Deep Dive

Universal Analytics is dead. Understand the event-based model of GA4.

  • Custom event creation and parameter passing.
  • Building Exploration Reports (Funnels, Pathing).
  • BigQuery Linking: Why raw data matters and how to set up the daily export.

3. Server-Side Tagging (sGTM)

With ad-blockers and iOS privacy changes, client-side tracking is degrading. We are shifting to server-side.

Key Concept: Instead of the user's browser sending data to Facebook/Google, it sends data to *our* cloud server, which cleans it and passes it along.

Module 3

Marketing Automation & Integrations

Connecting tools together using APIs and Webhooks to eliminate manual data entry.

1. Webhooks & APIs

The backbone of modern marketing stacks.

  • Webhooks: "Don't call us, we'll call you." How systems push real-time data to specific URLs.
  • REST APIs: Making GET and POST requests using tools like Postman to interact with our CRM.
  • Understanding Authentication (API Keys, OAuth 2.0).
Example cURL Request (POST)
curl -X POST https://api.hubapi.com/crm/v3/objects/contacts \
-H "Authorization: Bearer YOUR_TOKEN" \
-H "Content-Type: application/json" \
-d '{
"properties": {
"email": "test@example.com",
"firstname": "John"
}
}'

2. Advanced iPaaS (Make.com / n8n)

Moving beyond simple Zapier automations to build complex, conditional logic flows with error handling.

Module 4

Data Engineering for Marketers

Learn to query raw data and build dashboards that actually drive business decisions.

1. SQL for Marketers

You don't need to be a DBA, but you need to know how to query BigQuery to pull custom cohorts.

  • SELECT, FROM, WHERE, and GROUP BY.
  • Understanding JOINs (Inner vs Left) to combine CRM data with GA4 data.
  • Working with UNNEST() for GA4's nested event parameters.

2. Looker Studio Visualization

Stop sending screenshots of spreadsheets. Build automated, interactive reports.

  • Data blending limitations and workarounds.
  • Creating calculated fields using CASE statements and Regex.
Module 5

Web Fundamentals (Code for Marketers)

You don't need to build apps, but you need to know how the web works to fix frontend issues and implement custom tracking.

1. HTML / CSS

Understanding the DOM (Document Object Model).

  • Classes vs IDs (and why this matters for GTM).
  • CSS Selectors (e.g., div.header > a.btn).
  • Basic inline styling vs stylesheets.

2. JavaScript Basics

Reading, understanding, and writing basic JS for marketing implementations.

  • Variables (const, let).
  • Arrays and Objects.
  • Listening to DOM events (clicks, form submits).
  • Using the Browser Console for debugging errors.

Practice Challenges

Test your skills in real-world scenarios. No grades, just practical application.

Mini-Scenario

The Regex Redirect Mapping

Difficulty: Easy

We are migrating a blog. All old URLs structure `site.com/blog/2023/post-name` need to redirect to `site.com/resources/post-name`. Write the Regex matching rule and the target substitution.

Data Scenario

SQL Cohort Extraction

Difficulty: Medium

Download the `dummy_ecommerce.csv` file from the Sandbox. Write a SQL query (or use BigQuery sandbox) to find the total revenue generated by users whose `utm_source` was "newsletter" and who purchased more than 2 items.

Final Boss Project

The Broken Funnel Audit

Log into the WP Sandbox. The "Contact Us" form submission event is firing twice in GA4, the page has a massive CLS issue, and the JSON-LD schema is throwing a syntax error. Audit the page, fix the tracking via GTM, and write a 1-page report detailing the fixes.
